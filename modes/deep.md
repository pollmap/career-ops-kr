# Mode: deep

관심 공고의 회사/팀을 지원 전에 심층 조사한다. "지원할 가치가 있는가" 를 2~3페이지 리서치 보고서로 답한다.

## Purpose

- 사용자는 공고 단위가 아니라 **회사 단위**로 실패하는 경우가 많다 — 입사해도 3개월 뒤 후회하는 걸 미리 필터.
- 재무 건전성, 최근 뉴스, 퇴사자 리뷰, 연봉 밴드, Red flag 를 한 번에 묶어서 "지원 / 보류 / 기각" 을 권고한다.
- 읽기 전용. 권고는 사용자의 최종 판단을 돕기 위한 참고자료.

## Inputs

- 아래 중 하나:
  - `--company <name>` 회사명 (예: "Lambda256")
  - `--url <job_url>` 공고 URL → 회사 자동 추출
  - `--job-id <id>` SQLite JobRecord id
- 선택 인자:
  - `--depth <light|standard|deep>` 기본 `standard`
  - `--include-news-months <n>` 기본 12

## Process

1. **회사 기본 정보**
   - DART API (공시기업이면): 매출/영업이익/임직원수/설립일.
   - 공시기업이 아니면: `Nexus MCP > vault_*` 캐시 우선 → 실패 시 공식 홈페이지 회사소개만.
   - 투자 라운드: 해당되면 Crunchbase 공개 페이지 또는 벤처스퀘어/플래텀 기사 링크.
2. **최근 N개월 뉴스**
   - 네이버 뉴스 검색 (회사명 정확일치) → 최신 12건.
   - 제목만 수집, 본문 요약 금지. 링크+날짜+출처만.
3. **팀 구성 조사 (반자동)**
   - LinkedIn 조사는 **수동** — 프롬프트에 "LinkedIn에서 '{회사명} + 직무' 검색 후 5명 샘플 기록" 체크리스트만 제공.
   - 자동 스크래핑 금지 (LinkedIn ToS).
4. **퇴사자 리뷰**
   - 잡플래닛 / 블라인드 공개 페이지에서 별점 + 최근 리뷰 3건만 인용.
   - 로그인 필요 페이지 접근 금지. 공개 스니펫만.
5. **연봉 밴드**
   - 잡플래닛 공개 연봉 페이지 → 신입/경력 범위만.
   - 값이 없으면 "미공개" 로 명시. 추정 금지.
6. **Red flag 체크리스트**
   - 최근 12개월 내 대규모 감원 뉴스
   - 임금 체불 관련 뉴스/리뷰 키워드
   - 대표 교체 / 상장 폐지 위험
   - 주소 불명 / 실체 불분명 (T4~T5 공고에 특히 유의)
   - 경쟁사 대비 연봉 -20% 이하
7. **권장 판정**
   - "지원 권장 / 보류 / 기각" 중 하나 + 1문장 근거.
   - 데이터가 부족하면 "데이터 부족 — 수동 조사 필요" 로 표기. 점수 조작 금지.

## Output format

**대상 공고:** <JobRecord.title> (<source_url>)

## 1. 회사 기본
| 항목 | 값 | 출처 |

| 설립 | 2018 | DART |

  - ...

## 실데이터 원칙

- 웹 리서치 실패(네이버 차단/DART 500 등) → 해당 섹션에 **"미확인"** 명시. 값 조작/생성 금지.
- 공개 스니펫만 인용 — 로그인/결제가 필요한 페이지 접근 시도 금지.
- 모든 인용에 출처 URL + 접근 날짜 표기.
- 개인 SNS(개인 프로필 페이지) 접근 금지.

## HITL

- 없음. 리서치 리포트는 참고용.
- LinkedIn 직접 조사 섹션은 사용자가 수동으로 완성.

## Failure handling

- DART API 실패 → 재무 섹션 "미확인", 다른 섹션은 계속 진행.
- 네이버 뉴스 차단 → 뉴스 섹션 건너뛰고 경고.
- 회사명 모호 (동명 이회사) → 후보 리스트 출력 후 중단, 사용자가 `--company` 로 구체적 지정 요구.
- 결과물은 `reports/deep_<회사명>_<date>.md` 저장.
