# Mode: interview-prep

공고별 맞춤형 면접 답변 초안을 STAR + Reflection 구조로 생성한다. 한국 기업 문화의 겸손/공손 톤을 유지하며, 찬희의 실제 경험만 사용한다.

## Purpose

- 공고 1건에 대한 면접 준비 패키지 생성.
- 핵심: "이 공고에서 물어볼 법한 질문 + 찬희 경험 기반 답변 초안".
- **절대 날조 금지** — 찬희 `_profile.md` / `cv.md`에 근거 없는 경험 서술 금지.
- 결과물: `output/interview_<org>_<YYYYMMDD>.md` (markdown 리포트).

## Inputs

- `target_job`: `JobRecord` (org, title, raw_description, archetype)
- `modes/_profile.md` — 찬희 narrative (강점, 성과, proof points)
- `cv.md` — 이력서
- 선택 인자:
  - `--questions <n>` : 커버할 질문 수 (기본 8)
  - `--depth standard|deep` : 답변 분량 (기본 standard, deep은 질문당 300자+)

## Pre-check

- `_profile.md`에 경험 블록이 **3개 이상** 있어야 함.
- 부족하면 즉시 중단: "면접 준비를 위해 `_profile.md`에 성과/실패/협업 경험을 최소 3개 기록해주세요."

## Process

1. **회사 딥 리서치 훅** — `target_job.org` 기반 공개 정보 확인 포인트 나열:
   - 최근 보도자료/채용공지 타이틀 (크롤링 가능하면 실제 링크, 아니면 "수동 확인 필요"로 명시)
   - 사업 영역 / 경쟁사 / 최근 이슈
   - **없는 정보는 절대 지어내지 않는다.** "찬희가 직접 확인할 것" 섹션으로 분리.
2. **질문 유형별 초안 생성** (최소 커버):
   | 유형 | 예시 질문 |
   |------|-----------|
   | 자기소개 | 1분 자기소개 |
   | 지원 동기 | 왜 우리 회사/이 직무인가 |
   | 강점 | 본인의 강점 1~2가지 |
   | 약점 | 본인의 약점과 개선 노력 |
   | 성과 경험 | 가장 큰 성취 |
   | 실패 경험 | 실패와 배운 점 |
   | 협업/갈등 | 팀 갈등 해결 경험 |
   | Tricky | 공고 맞춤 기술/도메인 질문 (archetype 기반) |
3. **STAR + Reflection 구조** — 각 답변 초안은 아래 4블록으로:
   - **S**ituation: 당시 배경 (언제/어디서)
   - **T**ask: 맡은 역할 / 목표
   - **A**ction: 실제로 한 행동 (구체적 수치/도구/의사결정)
   - **R**esult: 결과 (숫자 우선, 없으면 정성적 명시)
   - **Reflection**: 배운 점, 성장 포인트, 이 공고와의 연결 (1~2문장)
4. **한국 비즈니스 톤 조정**:
   - 겸손한 자기 표현 ("제가 주도했습니다" → "팀과 함께 주도적으로 기여했습니다")
   - "~것 같습니다" 남발 금지 → 사실은 단정, 의견은 완곡
   - 영어 남용 금지 (전문용어만 예외)
   - 존댓말 일관성
5. **Tricky / 기술 질문** — `archetype` 기반 5개:
   - `BLOCKCHAIN` → "DID/ERC 표준 차이", "스마트컨트랙트 보안 이슈"
   - `RESEARCH` → "본인 종목분석 보고서 방법론", "밸류에이션 선택 이유"
   - `FINANCIAL_IT` → "코어뱅킹 아키텍처 이해도"
   - 각 질문 옆에 찬희 실제 답변 근거(`_profile.md` 레퍼런스) 또는 "학습 필요" 태그.
6. **공고별 커스텀 섹션** — "이 회사/직무만의 예상 질문 3개" + 답변 초안.

## Output

Markdown 리포트 구조:

```markdown
# 면접 준비: {org} — {title}

## 공고 핵심 (요약 3줄)
## 회사 딥 리서치 체크리스트 (찬희 수동 확인 항목)
## 예상 질문 & 답변 초안
### Q1. 자기소개
**의도:** ...
**STAR 답변 초안:**
- S: ...
- T: ...
- A: ...
- R: ...
- Reflection: ...
**키워드 링크:** (공고에서 매칭된 단어)
...
## Tricky / 도메인 질문 ({archetype})
## 공고 커스텀 질문 Top 3
## 찬희 체크리스트 (면접 전날)
```

## HITL

- 별도 게이트 없음 (지원 제출이 아니므로 G5 해당 없음).
- 단, **경험 데이터 부족 시 중단**하고 찬희에게 `_profile.md` 보강 요청.

## Failure handling

- `_profile.md` 경험 블록 < 3개 → 중단 + 보강 가이드 출력.
- 회사 공개 정보 접근 실패 → "수동 확인 필요" 섹션에 공란 TODO로 남김. 날조 금지.
- 파일 쓰기 실패 → UTF-8 인코딩 확인 후 재시도. `pathlib.Path` 사용.

## 실데이터 원칙

- 찬희가 실제로 하지 않은 프로젝트/성과를 답변에 넣지 않는다.
- 숫자 성과(매출, 사용자 수 등)는 `_profile.md` / `cv.md`에 있는 것만 사용.
- 가상 시나리오로 답변 만드는 것 금지 — 없으면 "이 질문은 찬희 경험 부족, 추가 기록 필요" 태그.
