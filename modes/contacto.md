# Mode: contacto

관심 회사의 현직자에게 **한국식 커피챗/정보성 미팅 요청** 메시지를 초안한다. 링크드인/사람인/이메일 채널 각각에 맞춘 150~250자 메시지를 생성한다.

## Purpose

- 공개 공고가 뜨기 전에 내부자 루트를 열어 두는 게 한국 취업시장의 현실.
- 찬희의 프로필(CUFA 회장 + MCP 서버 운영 + 퀀트 백테스트)을 **짧고 공손하게** 제시하면서 구체적 질문 1~2개로 부담 없이 대화를 요청한다.
- 메시지 **초안만** 생성. 실제 발송은 찬희 수동 (HITL G5-adjacent).

## Inputs

- 필수:
  - `--company <name>` 타겟 회사명
  - `--role <title>` 타겟 직무 (예: "블록체인 개발", "디지털자산 리서치")
- 선택:
  - `--connection <1st|2nd|cold>` 기본 `cold`
  - `--channel <linkedin|saramin|email|remember>` 기본 `linkedin`
  - `--language <ko|en>` 기본 `ko`
  - `--tone <formal|semi_formal>` 기본 `semi_formal` (한국 비즈 커피챗 관습)
  - `--question <q>` 찬희가 실제 궁금한 구체 질문 1개 (권장)

## Process

1. **톤 선택**
   - 1차 지인: `반말 금지 + 호칭 "선배님/대리님"`, 이모지 1개 허용.
   - 2차 연결: `semi-formal`, 공통 지인 1명 명시.
   - Cold: `formal`, 이모지 금지, 본인 소속/관심 선언 우선.
2. **자기소개 블록 (찬희 프로필 요약 — `config/profile.yml` 로드)**
   - 1줄: "충북대 경영 + CUFA 가치투자학회 회장 + Luxon AI 운영"
   - 기술 스택 1줄: "Python/SQL/MCP 서버 64개 운영 + 블록체인 리서치"
   - 보호 규칙: 학점/휴학/비전공 등 부정적 정보는 먼저 꺼내지 않음.
3. **질문 생성**
   - 찬희가 `--question` 제공 시 그대로 사용.
   - 없으면 archetype 기반 기본 질문 2개 제안 (사용자가 고름).
   - 질문은 구체적이고 답변자가 "5분 안에" 답할 수 있는 수준.
4. **메시지 작성**
   - 150~250자 목표 (한국어 기준, 공백 포함).
   - 구조: 인사 → 자기소개 1줄 → 관심/공감 1줄 → 질문 1~2개 → 마무리 인사.
   - 금칙어: "헤드헌팅", "추천", "채용 문의", "연봉" — 첫 메시지엔 절대 금지.
5. **출력**
   - 3개 버전 (길이/톤 다르게) 제시 → 찬희가 선택.
   - 각 버전 끝에 "발송 방법" 체크리스트.

## Output format

```markdown
# Coffee-chat draft — <회사명> / <직무>

**채널:** LinkedIn (cold)
**톤:** semi-formal
**글자수 목표:** 200~250자

## Version A (표준)
안녕하세요, 이찬희라고 합니다. 충북대 경영학부 재학 중이며 CUFA 가치투자학회 회장을 맡고 있습니다. <회사명>의 <구체적 프로덕트> 방향성에 관심이 있어 연락드립니다. 혹시 <질문>에 대해 5~10분 정도 편하게 말씀 나눠볼 수 있을까요? 바쁘실 텐데 무리되지 않는 선에서 언제든 편하신 시간 알려주시면 감사하겠습니다.

**글자수:** 218자 ✅

## Version B (짧게)
...

## Version C (긴 버전 — 추천서 포함)
...

## 발송 체크리스트
- [ ] 상대 프로필 최근 포스트 1개 읽고 맥락 맞추기
- [ ] 상대 현재 소속/직책 변경 없는지 확인
- [ ] 오탈자 검수
- [ ] 발송 후 7일 답변 없으면 추가 메시지 금지 (스팸 방지)
```

## HITL

- G5-adjacent: 메시지 초안까지만 생성. 실제 발송은 찬희 수동.
- 상대 개인정보(LinkedIn URL 등) 자동 저장 금지 — 해당 세션에서만 사용.

## Failure handling

- `config/profile.yml` 없음 → `_profile.md` 에서 fallback 수집. 둘 다 없으면 G1 온보딩으로 안내.
- `--company`/`--role` 한쪽 누락 → CLI가 즉시 중단, 나머지 인자 요구.
- 250자 초과 버전 → 자동 재작성 (한 번만). 그래도 초과면 경고 + 출력.
- LinkedIn/사람인 프로필 자동 스크래핑 금지 — ToS 위반.

## 실데이터 원칙

- 상대의 배경/학교/경력을 **추측해서 언급 금지**. 찬희가 미리 확인한 공개 정보만 사용.
- "저도 OO 출신입니다" 같은 거짓 공감 문구 금지.
- 회사 최근 뉴스 언급은 찬희가 수동으로 삽입 (`deep` 모드 결과 재사용 권장).
