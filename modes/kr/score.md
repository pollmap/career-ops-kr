# modes/kr/score.md — 한국 맞춤 스코어링 보정

> 디폴트 `modes/score.md` 의 A~F 그레이드 산출 결과(`default_score`)에 한국 금융권 특화 보정을 얹는다.
> 최종 점수는 0~100, 등급은 A(90+)/B(80~89)/C(70~79)/D(60~69)/F(<60).

## Purpose

- 한국식 연봉 구조를 실수령 기준으로 파싱.
- 찬희의 거주권(안산/서울/청주) 기준 위치 가중치 계산.
- 복리후생(재택/4일제/자율출근) 보너스.
- 기업 네임밸류 티어별 보정.

## 연봉 파서 (Korean compensation parser)

입력: 공고 본문 + JobRecord.
출력: `{base_krw: int, bonus_krw: int, total_krw: int, parse_confidence: float, source_note: str}`

**정규식 우선순위** (`modes/kr/_shared.md` 급여 구조 섹션 참조):

```python
# 1. 연 단위 명시
r'연봉\s*([0-9,]+)\s*(?:만원|만)'  → *10,000
r'연\s*([0-9,]+)\s*(?:만원|만)'   → *10,000

# 2. 월 단위 명시 → ×12 + 성과급 추정 15%
r'월\s*([0-9,]+)\s*(?:만원|만)'   → *12 *1.15

# 3. 기본급 + 성과급 분리
r'기본급\s*([0-9,]+)(?:만원|만).*성과급\s*별도'
r'기본급\s*([0-9,]+)(?:만원|만).*상여\s*([0-9]+)\s*%'

# 4. 범위
r'([0-9,]+)\s*[~∼–-]\s*([0-9,]+)\s*(?:만원|만)'  → 중간값 + warning

# 5. "초봉" / "신입 기준" → 하한선 확정
```

**주의**:
- 공고 통화 기본 KRW. 달러/엔 등 명시 시 스킵.
- 세전/세후 구분: 명시 없으면 세전(원천징수영수증 기준)으로 가정.
- "협의" / "회사 내규" → `parse_confidence = 0.0`, 점수 보정 없음.

## 연봉 점수 매핑 (신입 기준)

| 총액(세전) | KR score 보정 |
|---|---|
| 5,500만원 이상 | +15 |
| 4,800~5,499만원 | +12 |
| 4,200~4,799만원 | +8 |
| 3,800~4,199만원 | +5 |
| 3,300~3,799만원 | +2 |
| 3,000~3,299만원 | 0 |
| 2,700~2,999만원 | -3 |
| <2,700만원 | -8 (최저 수준) |

- 신입 금융권 평균 하한선 ≈ 3,000만원 (2026년 기준, 최저임금 210만원 ×12 = 2,520만원 상회 필수).

## 기업 네임밸류 티어

`modes/kr/scan.md` 부스트와 별개로 최종 점수에 반영:

| Tier | 기업 | 보정 | 안정성 |
|---|---|---|---|
| **S** | 5대 금융지주 (KB/신한/하나/우리/NH), 한국은행, 금감원 | +10 | 매우 높음 |
| **A** | 대형 증권 (미래/한투/삼성/NH/KB/신한투자), 핀테크 유니콘 (토스/카카오페이/카카오뱅크), 주요 자산운용 | +8 | 높음 |
| **B** | 중형 증권/운용사, 공공기관(예탁결제원/거래소), 블록체인 (두나무/빗썸) | +5 | 중간 |
| **C** | 카드/캐피탈/보험사 IT 자회사, 중형 핀테크 | +2 | 중간 |
| **D** | 저축은행/대출중개, 소형 핀테크 스타트업 | 0 | 낮음 |
| **F** | 검증 안된 토큰/DeFi 프로젝트, 불명확 법인 | -10 | 매우 낮음 |

**찬희 타겟 우선순위** (프로파일 기반): S ≥ A ≥ B > C > D >> F.

## Process

1. 디폴트 `modes/score.md` → `default_score` (0~100).
2. kr 보정 병렬 계산:
   - `parse_compensation()` → 연봉 보정
   - `compute_location_weight()` → 위치 보정
   - `parse_benefits()` → 복리후생 보너스
   - `classify_tier()` → 기업 티어 보정
   - `modes/kr/filter.md` 에서 전달된 `bonus_kr` + `cert_bonus` 합산
3. **최종 점수** = clamp(default_score + Σ(kr 보정), 0, 100)
4. **Cross-check**: default score 와 kr score 차이가 ±25 이상이면 `verify_flag = True` → HITL 확인 요청.
