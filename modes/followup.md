# Mode: followup

지원 후 후속 연락(follow-up) 이메일 초안을 상태별로 생성한다. 한국 비즈니스 메일 관례(정중/짧고/명확)를 따르며, **실제 발송은 사용자가 수동**으로 한다.

## Purpose

- 지원 N일 경과 / 면접 후 / 거절 통보 후 등 상태별 follow-up 메일 초안 작성.
- 사용자는 초안을 받아 직접 메일 클라이언트에 복사 후 발송.
- 자동 발송 금지.

## Inputs

- `target_job`: `JobRecord` (org, title, status, applied_at, interview_at, rejected_at)
- `status`: 현재 상태 (`applied`, `interview`, `rejected_self`, `rejected_site`, `offer` 등)
- `days_since_event`: 해당 상태 진입 후 경과일수
- 선택 인자:
  - `--recipient <name>` : 수신자 실명 (알면 입력, 모르면 "채용 담당자님")
  - `--language ko|en` : 메일 언어 (기본 ko)

## Process

1. **상태 + 경과일 매칭** — 아래 템플릿 중 선택:

| 상태 | 경과일 | 템플릿 |
|------|--------|--------|
| `applied` | 7~14일 | 지원 확인 정중 문의 |
| `applied` | 14~21일 | 재차 정중 문의 + 추가 자료 의사 표시 |
| `interview` | 1~2일 | 면접 감사 메일 |
| `interview` | 7~14일 | 면접 결과 정중 문의 |
| `rejected_self` (사용자 자발 거절) | 즉시 | 정중 사양 메일 |
| 외부 거절 통보 받음 | 1~3일 | 피드백 요청 (선택) |
| `offer` | 즉시 | 감사 + 검토 시간 요청 |

2. **한국 비즈니스 메일 톤 엄수**:
   - **제목 형식**: `[회사명] - {용건} - 지원자 이사용자`
   - **수신자 호칭**: "채용 담당자님께" / "{실명} 담당자님께" (실명 있으면)
   - **본문**:
     - 자기소개 1줄 (이름 + 지원 공고 + 지원일)
     - 용건 1~2문장 (장황하지 않게)
     - 감사 인사
   - **길이**: 본문 150~300자, 길어도 500자 이내.
   - **금지**:
     - 과도한 친근함 / 이모티콘
     - "~것 같은데" 추측 표현
     - 재촉하는 어조 ("아직 답변이 없어서")
     - 영어 남용 (전문용어 제외)
   - **필수**:
     - 정중한 마무리 ("좋은 하루 보내시기 바랍니다")
     - 서명 블록 (이름 / 연락처 / 이메일)

3. **상태별 핵심 메시지**:
   - **applied+7d**: 지원에 대한 검토 진행 여부 정중 문의, 추가 자료 필요 시 제공 의사.
   - **interview+1d**: 면접 기회 감사 + 기억나는 구체적 대화 1건 언급 (있으면).
   - **interview+7d**: 결과 진행 상황 정중 문의, 판단 지연 시 양해 표현.
   - **rejected (외부)**: 기회에 대한 감사 + 향후 개선을 위한 피드백 요청 (의무 아님 강조).
   - **offer**: 합격 감사 + 공식 조건 검토 위한 시간 요청 (정중한 "검토 후 회신").

4. **초안 출력** — 사용자가 바로 복사 가능한 형태:

```
제목: [{org}] - 지원 검토 문의 - 이사용자

{recipient}께

안녕하세요. {date}에 "{job_title}" 공고에 지원한 이사용자입니다.

{용건 본문}

바쁘신 와중에 검토 부탁드려 죄송하며, 좋은 하루 보내시기 바랍니다.
감사합니다.

이사용자 드림
{email} | {phone}
```

5. **사용자 발송 안내** — 수동 발송 체크리스트:
   - [ ] 수신자 주소 확인 (공고 원문 명시 이메일)
   - [ ] 오탈자 최종 검토
   - [ ] 첨부 파일 (필요 시)
   - [ ] 발송 후 `data/followup_log.jsonl`에 기록

## Output

- Markdown 파일: `output/followup_<org>_<status>_<YYYYMMDD>.md`
- 제목 + 본문 + 서명 + 발송 체크리스트

## HITL

- **G5 인접**: 자동 발송 절대 금지. 초안 생성만.
- 민감한 상황(offer 협상, 거절 피드백 요청)은 사용자 검토 후 수정 권장 안내.

## Failure handling

- `target_job`에 필수 필드 누락 시 (org/title/applied_at 등) → 에러, 초안 생성 중단.
- 수신자 이메일 불명 → 본문에 `[수신자 이메일 확인 필요]` placeholder로 표시.
- 공고 원문에 연락처 없음 → "수동 확인 필요" 태그. 가짜 이메일 생성 금지.

## 실데이터 원칙

- 사용자가 실제로 한 말/행동을 "면접에서 말씀드렸던..."처럼 지어내지 않는다.
- 면접 내용을 참조할 때는 사용자가 직접 구체화하도록 `{면접 구체 기억 — 사용자 수동 추가}` placeholder 사용.
